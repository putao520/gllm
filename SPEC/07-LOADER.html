2.4 Canonical Layout Normalisation (ARCH-LOADER-NORMALIZE)

SafeTensors / PyTorch 权重通常以 BF16 / F16 存储，且 nn.Linear.weight 的物理布局为 HF 风格 [out_features, in_features] row-major。Loader 在上传张量前必须完成布局归一化，同时保持原始 dtype 不做精度降级或升级：

关键常量:
  • 转置 tile size: TILE = 64 elements — 与 L1 cache line (64 B = 16 f32) 对齐、双维度 tile 在 L1 内可重用
    • 正确性要求:
  • Dtype 必须保持 bit-identical（BF16 权重不得升级为 F32，F16 权重不得升级为 F32）
  • Cache-blocked 转置必须与 naive 实现产生 bit-identical 输出（所有 finite 值）
  • 两者均需附带单元测试证明 bit-exactness（见 src/loader/mod.rs test 模块）
    • 禁止:
  • Load-time dtype 升级（如 BF16→F32），违反 ARCH-DTYPE-JIT-TYPED 原生混合精度设计
  • Naive 双层 for r × for c 转置（违反 ARCH-LOADER-CACHE-BLOCKED-TRANSPOSE）
  • 改变输出数值（不得以"SIMD fast-path"为借口破坏 bit-exactness）
    • 归一化
    输入
    输出
    并行化约束
    Dtype 保持
    BF16 / F16 / F32 / F64 原始字节
    原始 dtype 字节保持（BF16→BF16, F32→F32）
    dtype passthrough，零转换开销；JIT codegen 根据 TensorMeta.dtype 生成特化机器码（见 ARCH-DTYPE-JIT-TYPED）
    Linear 布局
    [out, in] row-major
    [in, out] row-major + meta.shape = [in, out]
    必须使用 cache-blocked (tiled) 转置；naive out[c * rows + r] = in[r * cols + c] 写步进为 rows * elem_size 字节（典型 1536 × 12288 ≈ 6 KB），每次 L1 miss，吞吐 50-200 MB/s；tiled 版本每个 64×64 tile 占 16 KB，读写双命中 L1，rayon 按列-tile 拆分 20 核并行

    铁律：SafeTensors loader 禁止将 _blocks + _scales 重打包为 GGUF 交错格式。

    说明

    2.5.4 GPU 性能考量

    分离格式对 GPU 后端至关重要：

    GPU MoE dispatch 的瓶颈是 memory bandwidth。交错格式的 stride=17 导致 warp 内每个 thread 的读取地址不对齐，memory transaction 利用率暴跌。分离格式的 stride=1 和 stride=16 都满足 coalescing 要求。

    ~100%
    交错
    17 (非 2 的幂)
    —
    严重撕裂
    ~7.5%

    2.5.5 GGUF 源的处理

    GGUF 文件内部以交错格式存储 MXFP4 数据。GGUF loader 负责在加载时拆分：

    ```

    GGUF file: [scale(1B) | nibbles(16B)] × num_blocks (stride=17)

    ↓ GGUF loader 拆分

    scales[]: [s0, s1, s2, ...] (stride=1)

    nibbles[]: [n0(16B), n1(16B), n2(16B), ...] (stride=16)

    ```

    拆分后的数据走与 SafeTensors 相同的后续路径。

    4.1 解析流程

    prost protobuf 解析 → Model/Graph/Node/Tensor 结构提取 → Graph Pattern Matching → 融合算子映射。

    4.2 Graph Pattern Matching

    三阶段流程：

    1. Matcher: 在 ONNX 图中匹配子图模式（如 MatMul → Add → Softmax → MatMul = Attention）

    2. Mapper: 将匹配的子图映射为 FusedKernel 描述（FlashAttention, SwiGLU 等）

    3. Kernel: 融合描述进入 JIT 编译管线，生成硬件原生代码

    不匹配的子图降级为 Atomic 算子（SPEC 授权 Fallback）。

    4.3 精度元数据

    从 ONNX tensor dtype 读取实际精度（F32/F16/INT8），禁止基于文件名推断（O1 真实性原则）。

    REQ-DTYPE-CHAIN-001 loader 层 raw-bytes passthrough 契约

    TensorProvider::load_weight(name) 必须返回 on-disk 原始 dtype 字节，禁止任何格式转换（BF16→F32、量化解包、精度扩展均禁止）。字节内容语义由 WeightBlob.dtype 独立携带，二者不得分离传递。

    验收标准：
    1. SafeTensors Provider: load_weight() 返回 WeightBlob { raw: Mmap{offset, len}, dtype: TensorView.dtype() 直接映射, source_quant: 浮点→Bf16/F16/F32; 量化→MXFP4/AWQ4 等 }，零拷贝无转换
    2. GGUF Provider (float): load_weight() 返回 WeightBlob { raw: Owned, dtype: TensorInfo.dtype, source_quant: Bf16/F16/F32 }，保留原始浮点格式
    3. GGUF Provider (quant): load_weight() 返回 WeightBlob { raw: Owned(raw_block_bytes), dtype: U8, source_quant: Q4_0/Q8_0 等 }，保留量化 block 原始字节
    4. ONNX Provider: load_weight() 返回 WeightBlob { raw: Owned(initializer_bytes), dtype: TensorProto.data_type 映射, source_quant: F32/F16/Bf16/F64 }，保留原始字节
    5. gllm Provider: load_weight() 返回 WeightBlob { raw: Mmap{offset, len}, dtype: gllm header dtype 字段, source_quant: TensorDir.quant_format 映射 }，零拷贝无转换
    6. grep -rn "convert_to_f32\|convert_tensor_to_f32\|bf16_to_f32" src/loader/ 在非测试代码中无结果，证明禁止加载时无条件转换到 F32
    7. WeightBlob.raw 字节内容与 on-disk 文件对应区域二进制一致（零拷贝路径 memcmp 验证）

    REQ-DTYPE-CHAIN-002 TensorMeta.dtype 必须反映 on-disk 格式，禁止推断覆盖

    SafeTensors/GGUF/ONNX/PyTorch 解析器填充 TensorMeta.dtype 时，必须直接映射文件格式中的 dtype 字段，禁止基于模型名/配置推断覆盖。用户主动覆盖仅允许通过 Loader::with_compute_dtype() API，且该覆盖只写 compute_dtype，不改变 TensorMeta.dtype（存储格式）。

    验收标准：
    1. SafeTensors 解析器: TensorMeta.dtype = TensorView.dtype() 直接映射，不读取 config.json 的 torch_dtype 字段覆盖
    2. GGUF 解析器: TensorMeta.dtype = TensorInfo.dtype 直接映射，GGUF 量化 tensor 映射为 DType::U8
    3. ONNX 解析器: TensorMeta.dtype = TensorProto.data_type 映射（FLOAT→F32, FLOAT16→F16, BFLOAT16→Bf16, DOUBLE→F64），不做推断
    4. PyTorch 解析器: TensorMeta.dtype = pickled tensor 的 dtype 字段映射，不使用 config.json 的 torch_dtype
    5. Loader::with_compute_dtype() 只设置 compute_dtype 字段，不修改已填充的 TensorMeta.dtype
    6. grep -rn "torch_dtype\|config.*dtype\|model_name.*dtype" src/loader/ 在 dtype 填充逻辑中无结果（证明不基于模型名/配置推断）
    7. 单元测试: 同一模型不同 torch_dtype 配置加载后 TensorMeta.dtype 一致（由文件格式决定，不受配置影响）

    REQ-DTYPE-CHAIN-003 WeightBlob 显式携带 dtype，禁止裸字节传权重

    loader 到 auto_graph/compile 之间的权重传输单元必须使用 WeightBlob { raw: WeightRaw, dtype: DType, shape: Vec<usize>, source_quant: QuantType }。禁止以裸 Vec<u8> / &[u8] / Cow<[u8]> 单独传递权重字节（必须配对 dtype）。WeightBlob.dtype 只读，构造后不可修改。

    验收标准：
    1. WeightBlob 结构体定义包含 raw: WeightRaw, dtype: DType, shape: Vec<usize>, source_quant: QuantType 四个字段，dtype 字段无 setter 方法
    2. auto_graph::build_compiler_graph() 接收 WeightBlob 而非裸字节，从 WeightBlob.dtype 提取 TensorMeta.dtype
    3. grep -rn "fn.*weight.*->.*Vec<u8>\|fn.*weight.*->.*&\[u8\]\|fn.*weight.*->.*Cow" src/loader/ src/arch/ src/graph/ 在权重传输接口中无裸字节返回类型
    4. WeightBlob 实现构造后 dtype 不可修改：编译期检查（dtype 字段为 pub 而非 pub mut，或通过 new() 构造后无 set_dtype）
    5. 量化 tensor: WeightBlob { dtype: U8, source_quant: Q4_0/AWQ4/... }，dtype 与 source_quant 联合解释字节语义
    6. 单元测试: 构造 WeightBlob 后尝试修改 dtype → 编译失败或运行时 panic

    WeightBlob

    字段类型必填约束说明
    dtypeenum[F32,F16,Bf16,U8,I32,I64,F64]是构造后不可修改，无 setter来自 on-disk 格式，只读。浮点: Bf16/F16/F32; 量化: U8; 其他: I32/I64/F64。量化 tensor 统一使用 U8，具体量化类型由 source_quant 标识
    shapearray<number>是元素级 shape，非字节级元素级 shape（非字节级），与 on-disk tensor shape 一致。量化 tensor 的 shape 为元素级（非 block 级）
    source_quantstring是浮点权重: Bf16/F16/F32; 量化权重: Q4_0/AWQ4/GPTQ4 等 QuantType variant原始量化/浮点类型标识。浮点权重标记为 Bf16/F16/F32；量化权重标记为具体 QuantType 变体（如 Q4_0, Q8_0, AWQ4, GPTQ4, MXFP4, NVFP4 等）
    rawblob是通过 Deref<Target=[u8]> 透明访问底层存储：mmap 零拷贝(SafeTensors/gllm) 或已分配内存(GGUF/ONNX)。通过 Deref 透明访问 raw bytes，字节内容语义由 dtype + source_quant 联合解释

    WeightRaw

    字段类型必填约束说明
    variantenum[Mmap,Owned]是Mmap: 零拷贝路径; Owned: 已分配内存路径存储变体: Mmap = mmap 零拷贝(Arc<Mmap> + offset + len); Owned = 已分配内存(Vec<u8>)
    mmap_handlejson否Mmap 变体必填Mmap 变体: Arc<memmap2::Mmap> 引用，支持同文件多 tensor 共享
    offsetinteger否Mmap 变体必填，≥0Mmap 变体: tensor 在 mmap 区域内的字节偏移。Owned 变体 offset=0
    owned_datablob否Owned 变体必填Owned 变体: 已分配的字节数据 Vec<u8>
    leninteger是>0原始字节长度，Deref<Target=[u8]> 可访问范围
