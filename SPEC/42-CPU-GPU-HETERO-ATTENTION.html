CPU-GPU 异构注意力协议 (ARCH-HETERO)

> 实现状态: 📋 设计完成 — REQ-HETERO-001~012 (编排层) + 关联扩展 REQ-KVOPT-011~013 / REQ-ARB-007~010 / REQ-WP-014~016 / REQ-MLA-010~012 / REQ-SYMDIM-011~013

定位

CPU-GPU 异构 1M 上下文推理架构的编排层 SPEC。协调 SPEC 12 (Strategy Arbiter)、SPEC 18 (SymDim/PagedKV)、SPEC 19 (KV Cache Optimization)、SPEC 21 (Weight Paging)、SPEC 33 (MLA Matrix Absorption) 五个 SPEC 的异构扩展，定义跨 SPEC 的协议约束和数据流契约。

> SSOT: 本 SPEC 定义异构注意力的编排协议。各子系统的具体实现约束在其所属 SPEC 中定义，本 SPEC 仅定义跨系统交互契约。

> 类比 SPEC 39 (统一编译器) 作为 SPEC 40 (数据流) 的前置，本 SPEC 作为各子系统异构扩展的前置编排层。

前置原则

1. 复用优先：StorageTier::CpuDram / ThreeTierSwapCoordinator / ExpertWeightPrefetcher / ExpertThermalManager / SwapInWorker / dma_helpers.rs 均已实现，异构扩展是增量而非重写。

2. ARCH-RUST-IS-CODEGEN 不冲突：Block Indexer 在 build_batch 阶段执行（Rust 调度器层），不属于推理热路径。Indexer 产出 position list 传入 Mega-Kernel ABI，JIT 代码只消费结果。

3. SchedulingMode 不可变：运行时动态切换通过 Strategy Arbiter 重新 arbitrate + Tier 迁移实现，不是切换 SchedulingMode。

4. 精度声明：MLA 吸收精度损失 = 0；两级 Block Indexer 漏检概率 < 1e-6，总精度损失 < FP8 量化误差 (~1e-3)；其余优化（Index 缓存/权重分页/KV DDR5/路由复用）位级精确。

设计决策

DD-1: 新增独立 SPEC 42 而非扩展现有 SPEC。异构注意力是跨 SPEC 的横切关注点，独立 SPEC 作为编排层避免循环依赖。

DD-2: 两级 Block Indexer 作为 SPEC 19 §8 新增章节。L1 粗筛消费 centroid 信号，L2 精筛消费 softmax_max 信号。

DD-3: Strategy Arbiter 扩展为 HeteroArbiterProfile，新增 4 个 StrategyBias 字段。不修改现有 arbitrate() 签名，新增 arbitrate_hetero() 重载。

DD-4: DDR5 KV 存储复用现有 StorageTier::CpuDram + ThreeTierSwapCoordinator，不新增 Tier。

DD-5: 权重分页异构扩展作为 SPEC 21 §7 新增章节，复用 ExpertWeightPrefetcher + ExpertThermalManager。

DD-6: MLA 吸收异构约束作为 SPEC 33 §6 新增章节，不改变数学模型。

DD-7: Block Indexer 在 CPU 侧执行，不违反 ARCH-RUST-IS-CODEGEN。

DD-8: IndexShare 每 4 层共享 indexer 作为 MlaConfig.index_share_interval 字段。

DD-9: PCIe 传输 latent[top-2048] 走现有 DMA/swap-in pipeline，新增 SelectiveSwapInRequest。

DD-10: 运行时动态切换通过 Strategy Arbiter 重新 arbitrate + swap model 实现，不违反 SchedulingMode 不可变铁律。

前置依赖

1. REQ-BACKEND-CAP-001~003 (BackendCapabilityModel)：异构路径选择需要知道每个设备能执行哪些操作。REQ-BACKEND-CAP 完成前，异构仲裁仅支持 CPU+GPU 通用路径（CPU 做 Indexer，GPU 做 Attention）。BackendCapabilityModel 提供的 capability flags：can_execute_attention、can_execute_indexer、supports_gpudirect_rdma 决定异构操作的具体分发策略。

2. REQ-ABI-HETERO-001 (Mega-Kernel ABI 扩展)：SPEC 40 当前 22 参数 L0 签名无 position_list 参数。新增 REQ-ABI-HETERO-001 定义 position_list 通过 KernelContext 共享内存协议传入 JIT 代码的机制，而非扩展 ABI 签名本身。选择共享内存协议而非 ABI 扩展的原因：(a) position_list 长度动态（top-2048），不适合固定 ABI slot；(b) 共享内存协议与现有 KernelContext 扩展点一致；(c) 零 ABI 破坏性变更。

风险评估

R-1: PCIe 带宽瓶颈（高风险）：8xB200 PCIe 5.0 x16 = 64 GB/s。DeepSeek V3 MLA 参数: d_c=512, d_rope=64, sizeof(f16)=2B → latent/position = 1152B。top-2048 positions × 4 layers (IndexShare) = 9.0MB/seq per share group。100 seq → 900MB → 14ms/次，为 10ms 目标的 1.4x。无 IndexShare 时每层 225MB → 3.4ms/层，61 层总计 207ms。缓解：IndexShare (4层共享 → 9.0MB/次) + GPUDirect RDMA + 异步双缓冲。

R-2: CPU Indexer 延迟（低风险）：DDR5-5600 ~80GB/s 读取 6MB < 0.1ms，CPU top-k 排序 < 1ms，总延迟 < 2ms。Batch=100 时需 Batch 融合（block representatives 只读一次）。

R-3: 模型参数依赖（中风险）：基于 DeepSeek V3/R1（已注册）作为参考模型。GLM-5.2 753B 作为未来扩展。

精度声明

  • MLA 吸收：数学等价变换，精度损失 = 0（SPEC 33 已验证 cos_sim >= 0.999）
  • 两级 Block Indexer：top-3 代表粗筛漏检概率 < 1e-6，总精度损失 < FP8 量化误差 (~1e-3)
  • Index 缓存：精确增量更新，精度损失 = 0
  • 权重分页/KV DDR5：PCIe 传输位级精确，精度损失 = 0
  • 路由复用：每次都算 Gate，精度损失 = 0
  • FP8 KV：与官方一致，精度损失 = FP8 量化误差
  • Q8 权重：精度 >= FP8

数据结构

接口契约

数据流

异构推理 decode step 完整数据流：

  1. build_batch: CPU Block Indexer 扫描 DDR5 上的 latent → top-2048 position list
  2. build_batch: Strategy Arbiter arbitrate_hetero() → HeteroArbiterProfile 决策
  3. SelectiveSwapIn: DMA 传输 latent[top-2048] DDR5 → GPU HBM
  4. Expert Prefetch: PCIe 带宽感知预取 hot expert weights → GPU HBM
  5. Mega-Kernel: GPU 侧 MLA absorbed attention + MoE FFN
  6. MLA absorption: W_absorb 预计算在 GPU, K_rope up-projection 在 GPU
  7. 输出: logits → 采样 → next token

实施顺序

  1. Phase 1: HeteroArbiterProfile + StrategyBias 扩展 (SPEC 12)
  2. Phase 2: DDR5 KV page 管理 (SPEC 18)
  3. Phase 3: 两级 Block Indexer (SPEC 19)
  4. Phase 4: Expert Offloading 异构协议 (SPEC 21)
  5. Phase 5: MLA 吸收异构约束 (SPEC 33)
  6. Phase 6: SelectiveSwapIn + PCIe 传输流水线 (本 SPEC)
  7. Phase 7: 端到端集成验证

验证

  • Block Indexer 精度: top-3 代表粗筛 vs 全量精确注意力，漏检率 < 1e-6
  • PCIe 传输正确性: latent DMA 传输后位级比对
  • 异构端到端: DeepSeek V3 1M 上下文异构推理 vs 纯 GPU 基线，输出 KL 散度 < FP8 量化误差
  • 延迟: Indexer + DMA 传输 + Attention 总延迟 < 20ms (decode step)

REQ-HETERO-004 两级 Block Indexer 协议

定义两级 Block Indexer 的跨 SPEC 协议约束。L1 粗筛：对 block representatives (centroid) 计算注意力得分，选取 top-64 blocks；L2 精筛：对 L1 选中的 blocks 内 sub-blocks 计算精确注意力得分 (softmax_max)，选取 top-2048 positions。Indexer 在 CPU 侧 build_batch 阶段执行，产出 position list 传入 Mega-Kernel ABI。

L1 粗筛消费 SPEC 19 定义的 centroid 信号 (KvPageHeader.centroid)
L2 精筛消费 SPEC 19 定义的 softmax_max 信号 (KvPageHeader.softmax_max)
Indexer 在 build_batch 阶段执行，不在推理热路径
top-3 代表粗筛漏检概率 < 1e-6
Indexer 产出的 position list 通过 KernelContext 共享内存协议传入 JIT 代码（见 REQ-ABI-HETERO-001，SPEC 40 扩展）

REQ-HETERO-007 PCIe 传输流水线协议

TC-HETERO-001: HeteroArbiterProfile 构造与拓扑描述

Arrange: 构建包含 x86_64 CPU + Nvidia GPU + CpuDram DDR5 的设备拓扑。

Act: 调用 HeteroArbiterProfile::new() 构造实例，验证字段赋值。

Assert: cpu_device == x86_64, gpu_device == Nvidia, ddr5_tier == CpuDram, pcie_bandwidth_gbs > 0, cpu_dram_bandwidth_gbs > 0。单设备构造返回 None 或退化配置。

TC-HETERO-002: StrategyBias 异构扩展字段默认值

Arrange: 构造 StrategyBias 默认实例（无 HeteroArbiterProfile）。

Act: 读取 4 个异构字段值。

Assert: kv_offload_preference == Adaptive, weight_offload_preference == Adaptive, indexer_device_preference == Cpu, hetero_batch_scale == 1.0。arbitrate() 行为与无异构字段时完全一致。

TC-HETERO-003: arbitrate_hetero() 单设备退化

Arrange: 构造 HeteroArbiterProfile 单设备拓扑（仅 GPU，无 CPU DDR5 tier）。

Act: 调用 arbitrate_hetero()。

Assert: 输出 ArbiterDecision 与 arbitrate() 完全一致（kv_offload_target/weight_offload_target/indexer_device 均指向 GPU）。

TC-HETERO-004: Block Indexer 两级筛选精度

Arrange: 构造 1M position KV cache（DeepSeek V3 参数），含 centroid/softmax_max 信号。

Act: 执行 L1 粗筛 (top-64 blocks) + L2 精筛 (top-2048 positions)。

Assert: top-3 代表粗筛 vs 全量精确注意力漏检率 < 1e-6；L1 选中 blocks 包含全量 top-64 blocks 的比例 > 99.9%。

TC-HETERO-005: IndexShare 跨层共享正确性

Arrange: 配置 MlaConfig.index_share_interval = Some(4)，61 层。

Act: 执行 indexer 16 次（61/4 向上取整），每 4 层共享结果。

Assert: 共享组内各层 position list 一致；IndexShare 模式 vs 逐层独立模式，KL 散度 < 1e-6；PCIe 传输量 = 9.0MB * 16 次 vs 2.25MB * 61 次（减少 ~38%）。

TC-HETERO-006: SelectiveSwapIn DMA 传输位级正确性

Arrange: DDR5 侧准备 latent 数据（f16），构造 SelectiveSwapInRequest (top-2048 positions)。

Act: 执行 SelectiveSwapIn DMA 传输到 GPU HBM。

Assert: GPU HBM 中接收的数据与 DDR5 源数据逐位比对，0 bit error。completion_callback 信号正确触发。

TC-HETERO-007: PCIe 带宽预算校验

Arrange: 配置 HeteroArbiterProfile.pcie_bandwidth_gbs = 64.0，batch_size=100，topk=2048。

Act: 调用 arbitrate_hetero() 计算带宽预算。

Assert: bandwidth_required = 9.0MB * 100 / indexer_interval = 900MB / 4 = 225MB/s。当 bandwidth_required > available 时返回 BandwidthInsufficient 决策。异步双缓冲 preload_next_layer=true 时，有效延迟降低约 50%。

TC-HETERO-008: DDR5 KV 页管理复用验证

Arrange: 构造 ThreeTierSwapCoordinator 实例，KV 页位于 CpuDram。

Act: 执行 GpuHbm→CpuDram 页迁移（复用现有 pipeline）。

Assert: 迁移后 CpuDram 页数据完整；DDR5 带宽感知优先级生效（DDR5-5600 页优先于 DDR4-3200）；latent page 按 d_c 维度连续存储。

TC-HETERO-009: Expert Offloading 三级放置正确性

Arrange: 配置 ExpertWeightPrefetcher + ExpertThermalManager，MoE 模型 256 experts。

Act: 异构仲裁后执行三级放置：hot→GPU HBM, warm→DDR5, cold→NVMe。

Assert: hot expert 在 GPU HBM 中可直接访问；warm expert 预取延迟 < PCIe 带宽预算；cold expert 仅在 LRU 驱逐后加载。DMA 通道无冲突（SelectiveSwapIn 与 Expert 预取分时复用）。

TC-HETERO-010: MLA 吸收异构数值对齐

Arrange: DeepSeek V3 MLA 权重，W_absorb 在 GPU 预计算，K_rope up-projection 在 GPU 执行。

Act: 异构路径 (latent DDR5→GPU + GPU absorbed attention) vs 纯 GPU 基线。

Assert: cos_sim >= 0.999（与 SPEC 33 一致）；QK^T 计算结果位级一致（相同 GPU 代码路径）。

TC-HETERO-011: 动态 Tier 迁移压力阈值触发

Arrange: 配置 memory_pressure threshold = 0.85（由 arbitrate_hetero() 输出），HBM 使用率逐步上升。

Act: memory_pressure 超过 0.85 时触发 GpuHbm→CpuDram 迁移。

Assert: 迁移成功后 memory_pressure 降至阈值以下；SchedulingMode 未被修改；ThreeTierSwapCoordinator pipeline 正确执行。

TC-HETERO-012: 异构端到端验证

Arrange: DeepSeek V3/R1 模型，1M 上下文，异构配置 (CPU DDR5 + GPU HBM)。

Act: 执行完整异构推理（Indexer + DMA + Attention），对比纯 GPU 基线输出。

Assert: (1) Block Indexer 精度：漏检率 < 1e-6；(2) PCIe DMA 传输：0 bit error；(3) KL 散度 < FP8 量化误差 (~1e-3)；(4) Indexer + DMA + Attention 总延迟 < 20ms (decode step, single sequence)。
