REQ-C1-001 c1 DialogueGate Intent Tracker — granite encoder + 3 层 gated cell 状态递推

c1 v2 对话意图追踪模型：encoder 使用 granite-embedding-311m (ModernBERT，冻结，外部预编码为 e_t(768))，tracker 为 3 层 stacked DialogueGateCell 跨 turn 状态递推。DialogueGateCell 递推：h'=α*h+gate*candidate，gate/α/candidate 由当前 turn 输入 + 上一轮隐状态经线性投影 + sigmoid/tanh 激活得到。输出经 label-query attention head 产出 7-intent + 3-difficulty 标注。

验收标准：
  1. encoder 复用 granite ModernBERT backbone，tracker 为 3 层 DialogueGateCell 单步递推图
  2. 递推公式 h'=α*h+gate*candidate 在图算子中显式表达，α/gate/candidate 各自独立投影
  3. 跨 turn 状态 (h_0..h_2) 由调用者管理，tracker.step(token_hidden, prev_state) → (intent7, diff3, new_state)
  4. 权重从 v2_granite_best.pt (47 keys) 加载，tensor name 映射到图规范名

REQ-C1-002 c1 label-query attention head — intent7 + difficulty3 输出

c1 tracker 输出头：label-query attention 机制，以可学习 label query 对 dialogue gate 输出做注意力加权，产出 7 维 intent logits + 3 维 difficulty logits。intent 7 类覆盖核心对话意图，difficulty 3 类标注任务难度。head 权重与 tracker 共同加载，前向时与 gate 输出做点积注意力。

验收标准：
  1. label-query attention head 输出维度 = intent(7) + difficulty(3)
  2. head 权重从同一 .pt 文件加载，与 DialogueGateCell 权重解耦
  3. 前向路径在 C1V2Tracker::heads_forward 中真实实现，非 stub

REQ-QGUARD-001 Qwen3Guard-Stream backbone — 复用 Qwen3-0.6B decoder

Qwen3Guard-Stream-0.6B 模型架构：backbone 复用 Qwen3-0.6B (gllm 已支持的 qwen3 模板)，架构注册表新增 qwen3_guard_stream token，family=decoder，复用 qwen3 backbone 权重拓扑与 KV cache 机制。backbone 前向产出 per-token hidden state，喂给 guard 分类头。

验收标准：
  1. arch/registry.rs 注册 qwen3_guard_stream token，canonical 复用 qwen3 backbone
  2. backbone 权重加载复用 qwen3 safetensors loader，dtype 从 TensorMeta 推断 (ARCH-JIT-DATA-YIELDS)
  3. backbone forward 路径与 qwen3 生成模型一致，仅输出 head 不同

REQ-QGUARD-002 REQ-QGUARD-002 Qwen3Guard per-token 多类分类头 — 2路径×4头 (risk3+cat8+qrisk3+qcat9)

Qwen3Guard 分类头 (qwen3_guard.rs Qwen3GuardHead)：per-token 多类分类，非单一 sigmoid score。双路径拓扑 (response + query)，每路径含 Linear(hidden→guard_inner=512, no bias) → RMSNorm(guard_inner) → 2 个并行头。response 路径: risk_level_head(512→3) + category_head(512→8)。query 路径: query_risk_level_head(512→3) + query_category_head(512→9)。8 个权重张量: risk_level_category_pre/risk_level_category_layernorm/risk_level_head/category_head + query_risk_level_category_pre/query_risk_level_category_layernorm/query_risk_level_head/query_category_head。moderate_token(hidden[1024]) → GuardModerationResult{risk_level_logits[3], category_logits[8], query_risk_level_logits[3], query_category_logits[9]} (共 23 logits)。

验收标准：
  1. MultiClassGuardProbe 支持 risk(3) + category(8/9) 多类输出，非单 score sigmoid
  2. 权重从 safetensors 提取 risk_weight/risk_bias/category_weight/category_bias，dtype 顺从权重文件
  3. classify(hidden) 返回 (risk_logits, category_logits) 二元组，调用者做 argmax/softmax
  4. 扩展 GuardProbe 而非替换，现有 GuardProbeWeights (单 score) 保持兼容

REQ-QGUARD-003 Qwen3Guard 流式 stream_moderate API — 跨 token 增量复用 KV cache

stream_moderate 流式审核 API：StreamModerateBuilder 接收逐 token 输入，跨 token 维护 stream_state，复用 backbone KV cache 增量机制。每个 token 调用 stream_moderate(token, role, stream_state) → (risk, category)，stream_state 由调用者管理生命周期。role 区分 query/response 决定 category 类别数 (9 vs 8)。

验收标准：
  1. StreamModerateBuilder::stream_moderate(token, role, stream_state) → (risk, category) per-token 输出
  2. stream_state 跨 token 存活，复用 backbone KV cache 增量，禁止每 token 重算全序列
  3. role=Query → category 9 类 (含 Jailbreak)，role=Response → category 8 类
  4. API 在 classify.rs 实现，backbone 前向走 qwen3 路径，guard head 走 MultiClassGuardProbe
