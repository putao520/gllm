# AI 架构与开发核心指导纲领 (AI Development Master Guidelines)

**本文档是专为参与 `gllm` 开发的 AI Agent 以及高级核心开发者准备的架构底线守则。在撰写任何代码、重构内核或执行系统设计之前，必须通读并严格遵守以下准则。**

## 1. 原生态零妥协 (Zero-Compromise Purity)
- **禁止外挂语言边界**: 无论是调度逻辑还是编译逻辑，所有行为必须被内聚至纯 Rust 环境。无论遇到何种瓶颈，绝不允许通过 FFI 调用 Python 黑盒或其它沉重的运行时环境。
- **强制的物理特化编译**: 虽然计算图在顶层被静态全模图封存，但底层的单个核函数（Kernels）在承接尺寸变换（如 Head-Dim 或 Sequence-Length 边界）时，在设计上必须坚决利用 JIT (如 PTX 生成器) 实现**基于静态 DType 特化前提下的现场动态形状映射**。不再做维度相关的汇编软路由（TurboQuant 原则）。遇到非标的 112 维度计算时，坚决禁止退化进行强行 Padding 补 0 对齐到 128！必须写出 100% 寄存器命中率生成的指令流。
- **TurboQuant 运行时精度优化**: gllm 支持加载 SafeTensors/GGUF/ONNX 全格式权重文件。QuantType 直接驱动 JIT 生成硬件原生内核，推理过程中无类型判断分支。TurboQuant 是 gllm 在前向传播中执行的运行时数学优化（3 个在线 FWHT 旋转 + KV 非对称量化 + RaBitQ 无偏修正），使推理精度逼近无损。详见 02-ARCHITECTURE.md §11。

## 2. 彻底的零额外损耗 (Zero-Overhead Freeloading)
- **绝对的“白嫖”法则 (Piggybacking)**: 当你需要采集一个模型的数学特征（例如由于激活动态量化需要 Max、Min，或者由于稀疏裁剪需要 L2 范数边界）时，**绝对禁止单独开启一个内存读取扫描循环**。所有的特征统计逻辑必须无开销地、如寄生般融合缝合进上游必定需要执行的窄口路径核函数中（例如 `RmsNorm` 的方差规约循环，或 `KV-Scatter` 的物理写出队列中）。
- **核心数据结构的极致统一 (Unified Caching)**: MoE 的专家表重（Expert Weights）和高维度动态内存缓存（KV Cache），在下层物理映射上必须被剥去表层含义，视为极其统一的物理概念块：`Virtual Page`。禁止为两个同样面临置换调度（Swap/Drop）的对象编写互相孤立的 Manager/LRU 维护代码。

## 3. 硬件设备端的智能转移 (Device-side Control Flow)
- **去 CPU 控制面板化**: 在复杂的推流和 Attention 动态执行流期间，不要让 GPU 不断休眠以等待 CPU 传递“哪些需要算、哪些跳过”的决策。将大批量条件判断全部通过底层编译器转换为 JIT 直出的 `if-else / Loop` 硬件指令跳板，交由硬件的分支预测器定夺。主机调度 Executor 在微观回合只能发射大型单图。
- **Mega-Kernel 路由同步 (In-Kernel Routing)**: 彻底抛弃 CPU 端的多线程环形队列（RingBuffer）或任何无锁消息同步循环。工作交接、条件分支与状态回写必须全部寄托在 GPU 内部物理单元之间以原子操作直接对戳。在服务端架构彻底肃杀需要唤醒或抢占的异构线程交互开销！

## 4. 彻底的精确控制与死板的验证体系 (Absolute Accuracy)
- **杜绝静默性能降级 (No Silent Performance Collapse)**: 所有被要求触及极致底层的高级性能内核，在遇到如环境缺失等硬性异常情况，应用干脆的 `Result::Err` 反击。坚决不允许暗箱内敛退步并隐式替换为性能下降极大却不报错的标量函数。一旦发生吞吐异常，必须在开发期暴露出错误以根绝。
- **依赖系统级别回收规则 (Ownership is Law)**: 上下文记忆表、全局公共 Radix 树等极度泛化的网络数据分布映射图，彻底结合且仅依靠 Rust 强悍的物理层变量存活期管控（智能指针级的 `Arc`/`Drop` 作用域）。严禁搭建任何依赖计时器或定时轮询发起的外部垃圾回收算法。

---
> **Agent 读取效忠宣言**: 当 Agent 接收指令触碰系统底层引擎并准备产出代码前，必须确保前置方案验证没有触犯上述的 4 大守则“原罪”，以维持这台极致虚拟引擎最原初的凌厉感。
