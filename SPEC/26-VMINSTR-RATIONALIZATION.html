§1 去类型化合并 (REQ-VR-001~004)

将 VmInstr 中 dtype 硬编码在指令名中的变体（GGUF 加载 11 个、DotProduct 5 个、GPR 操作 11 个、Quant* 解码 18 个）合并为参数化指令，参照 Triton TTIR 设计原则使 dtype 存在于类型签名中而非指令名中，将总变体数从 142 精简至约 97 个。

验收标准
  1. VmInstr 新增/合并变体编译通过: cargo check
  2. auto_select.rs 新增对应 match arm
  3. ISA lowering (x86_64/AArch64/GPU) 生成正确指令
  4. 单元测试: VmInstr round-trip 验证
§1.1 GGUF 量化块加载合并 (REQ-VR-015)

将 11 个 GgufXxxLoad 变体合并为 2 个参数化指令（QuantBlockLoad + QuantBiPlaneLoad），通过 BlockUnpackMode/BiPlaneMode 枚举描述解包策略，消除每新增一个 GGUF 量化格式就新增一个 VmInstr 变体的膨胀问题。

当前：11 个 GgufXxxLoad 变体，全部是 load bytes → unpack → dequant → compute_dtype，仅解包算法不同。compute_dtype 由 TensorMeta.dtype × DeviceProfile 推导，量化路径计算精度从权重 dtype 推导(多精度混合支持 F32/BF16/F16)，原生浮点路径保持原始精度。 目标：2 个参数化变体 + 保留 2 个 scale 加载。
验收标准

  2. auto_select.rs 新增对应 match arm
  3. ISA lowering (x86_64/AArch64/GPU) 生成正确指令
  4. 单元测试: VmInstr round-trip 验证
§1.1.1 QuantBlockLoad — 单平面量化块加载

```

QuantBlockLoad:

输入:

dst: VRegId — 目标 Vec 寄存器 (compute_dtype, 由 TensorMeta.dtype × DeviceProfile 推导)

base: VRegId — 基地址 GPR

offset: OffsetExpr — 字节偏移

unpack: BlockUnpackMode — 解包模式（完整描述 byte→i32→compute_dtype 策略）

width: SimdWidth — SIMD 宽度

dtype: QuantPrecision — 输出计算精度 (从 op_input_dtype 传播, 非 F32 硬编码)

BlockUnpackMode: — 替代 7 个 GgufXxxLoad 变体

| Int8 — i8 → compute_dtype 直转 (原 GgufInt8Load)

| F16Broadcast — f16 → compute_dtype + broadcast (原 GgufF16ScaleLoad)

| SignedNibbleLow — & 0x0F, sub 8.0 (原 GgufInt4Load)

| UnsignedNibbleLow — & 0x0F, no bias (原 GgufUInt4Load)

| SignedNibbleHigh — >> 4, sub 8.0 (原 GgufInt4HighLoad)

| UnsignedNibbleHigh — >> 4, no bias (原 GgufUInt4HighLoad)

| Bitpack2 — 2-bit ×4 per byte, sub bias (原 GgufInt2Load)

```

删除的变体：GgufInt8Load, GgufF16ScaleLoad, GgufInt4Load, GgufUInt4Load, GgufInt4HighLoad, GgufUInt4HighLoad, GgufInt2Load x86_lower 分发：按 unpack 值 match，内部逻辑不变（只是从 7 个 VmInstr match arm 变为 1 个 + 内部 match on BlockUnpackMode）。

```

QuantBiPlaneLoad: — 替代 3 个多平面 GgufXxxLoad 变体

输入:

qs_base: VRegId — 低 bit 平面基地址

extra_base: VRegId — 高 bit 平面基地址 (qh / hmask)

bias: f32 — 解码后减去的偏置 (恒为 f32, 量化偏置参数)

mode: BiPlaneMode — 合并策略

width: SimdWidth

dtype: QuantPrecision — 输出计算精度 (同 QuantBlockLoad.dtype)

```

    §1.7 SM 分区同步 VmInstr (SPEC 32 联动)

    SPEC 32 §2 SM 分区 Ping-Pong 使用硬件同步原语实现 decode∥prefill 并行。这些原语不在 VmInstr 层定义，而是在 GPU ISA lowering 层直接 emit。

    设计决策: SM 分区同步（cluster.sync / mbarrier / grid_sync）是物理 CTA 级操作，不经过 VmInstr 虚拟机。原因：

    1. 同步粒度是 CTA/warp 级，不是指令级——VmInstr 是指令级抽象

    2. 同步原语高度硬件特化（SM90 cluster.sync vs SM80 grid_sync vs SM70 ring barrier），无法用统一 VmInstr 表达

    3. SPEC 32 的 SM 分区在 mega_kernel_emit.rs 的 CTA 编排层实现，不在算子 lowering 层

    与现有 VmInstr 的关系:
  • CommBarrier（§1.6）是分布式通信同步（跨节点），保留为 VmInstr
  • SM 分区同步是单 GPU 内同步，在 JIT codegen 的 CTA 编排模板中直接 emit PTX/HIP 指令
  • CPU 路径使用 std::sync::Barrier（Rust std），不需要 VmInstr

    • > 注: 如果未来需要 JIT 内部更灵活的同步控制（如动态 barrier），可以扩展 VmInstr。当前设计中 SM 分区同步是编译时确定的 CTA 编排，不需要运行时 VmInstr。

    §2 缺失指令补全 (REQ-VR-005~010)

    补全 VmInstr 指令集中缺失的关键能力：向量 Shuffle/Permute、向量 Lane 提取/插入、向量常量加载、原子 CAS、GPR 一元操作（Popcount/CLZ/ByteSwap），以及 Mxfp4/Nvfp4 合并到 QuantBlockLoad，使 TraceOp 到 VmInstr 映射完整覆盖所有算子需求。

    数据结构

    ENT-OP

    OpKind — 算子种类枚举

    variantinput_countoutput_countdtypehead_dimeps
    字段类型必填约束说明
    string是OpKind enum算子变体名 (Gemm/RmsNorm/RoPE/Attention/Gather/MoEGate/etc.)
    number是≥1输入 tensor 数量
    number是≥1输出 tensor 数量
    string否QuantPrecision操作数据精度
    number否仅 Attention/RoPE/QkNorm注意力头维度
    number否仅 RmsNorm/LayerNorm/ValueNorm归一化 epsilon

    ENT-QUANT-PRECISION

    QuantPrecision — dtype精度枚举

    kindpackingblock_sizegroup_size
    字段类型必填约束说明
    string是F32/F16/BF16/TF32/FP8E4M3/FP8E5M2/FP6E2M3/FP6E3M2/FP4E2M1/INT8/INT4/INT2/INT1DTypeKind 基础精度类型
    string是Plain/MXBlock/GGML/GPTQ/AWQ/BitnetPackingFormat 存储打包方式
    number是0=不适用量化块大小
    number是0=不适用分组量化的组大小

    ENT-TRACE-OP

    TraceOp — SSA中间表示算子（enum TraceOp in gllm-kernels/src/compiler/trace.rs）

    variantinput_indexconst_valueleft_operandright_operand fma_third_operandreduce_kindquant_precision
    字段类型必填约束说明
    string是 Input|Const|Add|Sub|Mul|Div|Fma|Neg|Abs|Exp|Sqrt|Rsqrt|Tanh|Recip|Log|Sigmoid|Max|Min|ConditionalBranch|QuantFma|BlockScale|Cast|HReduce|Prefetch|NonTemporalStore|BitExtract|Permute枚举变体名
    number否u32, 仅 Input 变体加载第 i 个输入元素
    number否 f64, 仅 Const 变体浮点常量值
    number否ValueId, 二元运算左操作数Add/Sub/Mul/Div/Max/Min 的左操作数
    number否ValueId, 二元运算右操作数Add/Sub/Mul/Div/Max/Min 的右操作数
    number否ValueId, FMA 加数Fma(a,b,c) = a*b+c 的 c
    string否Sum|Max|Min, 仅 HReduce 变体水平归约操作类型
    string否QuantPrecision, 仅 QuantFma/Cast 变体混合精度 dtype

    ENT-VM-INSTR

    VmInstr — 虚拟指令集 (102+指令)

    variantdtypewidthdstsrcoffset basestep
    字段类型必填约束说明
    string是VecLoad|VecStore|Fma|LoopBegin|...指令变体名 (102+种)
    string是QuantPrecision数据精度类型
    string是SimdWidthSIMD宽度 (128/256/512/SVE)
    string否VRegId目标寄存器
    string否VRegId源寄存器
    string否OffsetExpr内存偏移表达式
    string否VRegId基址寄存器
    number否usize循环步长 (LoopBegin专用)
