统一编译器架构 (39-UNIFIED-MEGA-KERNEL)

> 核心定位: 本文档是统一编译器架构的入口索引。原 SPEC 39 混合了外部接口与内部实现细节，已按受众拆分为两个子文档。

§0.1 文档拆分

原 SPEC 39 已按受众拆分为：

  • 39A-COMPILER-CONTRACT — 接口契约（读者=gllm 开发者）
    • REQ-UMK-001: compile 入口签名
    • REQ-UMK-002: 输入类型 CompilerGraph
    • REQ-UMK-003: 输出类型 MegaKernelFn
    • REQ-UMK-004: 辅助小图（SG/Vision/Audio）
    • REQ-UMK-006: 图输出契约
    • REQ-UMK-011: 禁止规则
  • 39B-COMPILER-INTERNALS — 内部设计（读者=gllm-kernels 开发者）
    • REQ-UMK-008: 权重布局（内部决策）
    • REQ-UMK-012: GroupMarker 机制
    • REQ-UMK-013: ForwardPhaseDispatch 三路分支
    • REQ-UMK-015: LoweringContext 数据结构
    • REQ-UMK-016: EmitState 可变状态
    • REQ-UMK-017: 并行化策略（第一层）
    • REQ-UMK-018: 融合组级并行（第二层）
    • REQ-UMK-019: ISA Lowering

§0.2 拆分原因（A5-2 修复）

原 SPEC 39 混合了外部接口契约（gllm 开发者需要遵守）和内部实现细节（gllm-kernels 开发者需要遵守），导致不同受众难以快速定位相关信息。本次修复按接口 vs 内部拆分，提高文档可读性。

§0.3 xref 关系

39A 和 39B 之间存在双向 xref：

  • 39A 接口契约引用 39B 内部实现（如 39A REQ-UMK-002 输出类型引用 39B REQ-UMK-013 ForwardPhaseDispatch）
  • 39B 内部实现引用 39A 接口契约（如 39B REQ-UMK-012 GroupMarker 引用 39A REQ-UMK-001 compile 入口）

REQ-UMK-20 MegaKernelFn 22 参数 ABI — 编译产物统一调用签名

MegaKernelFn 统一 ABI 22 参数签名：所有编译产物使用相同的 22 参数 ABI，包括输入/输出 buffer、KV Cache 指针、SymDim 运行时参数、callback slot 等。编译器不假设图结构，ABI 是图拓扑的纯函数。

验收标准：
  1. 符合 MegaKernelFn 统一 ABI 22 参数签名：所有编译产物使用相同的 22 参数 ABI，包括输入/输出 buffer、KV Cache 指针、Sym 的约束定义

REQ-UMK-21 GraphTopologyAnalysis — 图拓扑驱动的编译器行为推导

GraphTopologyAnalysis 拓扑推导：从图 OpKind 存在性推导一切编译器行为——has_generate_loop/needs_persistent_kv_cache/output_sink/num_kv_tensors 等。替代 is_encoder/needs_kv_for_decode 等 bool 参数。

验收标准：
  1. 符合 GraphTopologyAnalysis 拓扑推导：从图 OpKind 存在性推导一切编译器行为——has_generate_loop/needs_persi 的约束定义

REQ-UMK-22 Phase 3 融合组顺序 emit — GroupMarker 替代硬编码层循环

Phase 3 按融合组顺序 emit：编译器按融合组（GroupMarker 标记的同构子结构）顺序生成代码，不硬编码执行阶段。融合组由 SemanticDAG 自动分析生成。

验收标准：
  1. 符合 Phase 3 按融合组顺序 emit：编译器按融合组（GroupMarker 标记的同构子结构）顺序生成代码，不硬编码执行阶段。融合组由 SemanticDA 的约束定义

REQ-UMK-23 ForwardPhaseDispatch — Argmax 存在时插入采样管线

ForwardPhaseDispatch 条件编译：仅当图含 Argmax 算子时插入 ForwardPhaseDispatch（采样管线）。无 Argmax 的图（Embedding/Reranker）不生成采样代码，零开销。

验收标准：
  1. 符合 ForwardPhaseDispatch 条件编译：仅当图含 Argmax 算子时插入 ForwardPhaseDispatch（采样管线）。无 Argmax 的约束定义

REQ-UMK-24 SharedKvRef GprCondAction — Gemma 4 条件跳过 KV op

SharedKvRef 条件分支（Gemma 4）：Consumer 层通过 GprCondAction 条件跳过 K/V 相关 5 个 op。条件 layer_idx < (num_layers - num_kv_shared_layers)。非 SharedKvRef 模型条件永远 true，零开销。

验收标准：
  1. 符合 SharedKvRef 条件分支（Gemma 4）：Consumer 层通过 GprCondAction 条件跳过 K/V 相关 5 个 op。条件 layer 的约束定义

REQ-UMK-25 编译管线四阶段 — Scalar→SemanticDAG→Fusion→ISA Lowering

编译管线四阶段完整流程：Phase 0 (Scalar+SymExec) → Phase 1 (SemanticDAG) → Phase 2 (Fusion+HW) → Phase 3 (ISA Lowering)。唯一入口 compile()，所有图走同一管线。

验收标准：
  1. 符合 编译管线四阶段完整流程：Phase 0 (Scalar+SymExec) → Phase 1 (SemanticDAG) → Phase 2 (Fusion+H 的约束定义

REQ-UMK-26 融合模式六种 — EpilogueInjection/LoopFusion/TileLevel/ComputeRoot/QkvShared/NormIntoGemm

融合模式六种：EpilogueInjection / LoopFusion / TileLevelFusion / ComputeRoot / QkvSharedInput / NormIntoGemm。融合在 Phase 2 执行，由 FusionRule 生成融合图，JIT codegen 根据 DeviceProfile 为每种硬件生成最优机器码。

验收标准：
  1. 符合 融合模式六种：EpilogueInjection / LoopFusion / TileLevelFusion / ComputeRoot / QkvShare 的约束定义

REQ-UMK-27 CPU/GPU 统一编译 — 共享 IR/GraphType + DeviceProfile 驱动 codegen

CPU/GPU 统一编译架构：CPU 和 GPU 共享完全一致的 CompilerGraph IR 和 GraphType，通过 DeviceProfile 驱动不同 codegen。禁止以 Cpu/Gpu 为前缀的 GraphType 变体。禁止 CPU 专用图构建器。

验收标准：
  1. 违反禁止项「禁止以 Cpu/Gpu 为前缀的 GraphType 变体」时编译/运行必须失败
  2. 违反禁止项「禁止CPU 专用图构建器」时编译/运行必须失败

REQ-UMK-28 JIT 缓存协议 — 三级缓存与 SymDim 动态维度无重编译

JIT 缓存协议：编译只发生在模型加载时，推理热路径禁止任何编译行为。缓存粒度 = MegaKernelFn（单次 compile() 产出）。动态维度通过 SymDim::Symbolic + ShapeBinding 运行时绑定，不触发重编译。三级缓存 L1(进程内)/L2(共享内存)/L3(磁盘)。

验收标准：
  1. 违反禁止项「禁止任何编译行为」时编译/运行必须失败

REQ-UMK-29 LazyAbiLoad 按需加载 — op lowering 按需 demand 替代前置 bool flag

LazyAbiLoad 按需加载：prologue 阶段不前置判断图需要哪些 ABI 指针，而是 op lowering 按需 demand()。遇到什么 op 就加载什么指针，未 demand 的指针不加载。替代 has_xxx/needs_xxx 前置 bool flag。

验收标准：
  1. 符合 LazyAbiLoad 按需加载：prologue 阶段不前置判断图需要哪些 ABI 指针，而是 op lowering 按需 demand()。遇到什么 op 的约束定义

REQ-UMK-30 SymDim 穿透 — Symbolic 维度编译时保留运行时绑定

SymDim 穿透编译时保留：SymDim::Symbolic 维度在整个编译管线中保持 Symbolic 语义，BoundExpr::Symbolic 从运行时参数读值。禁止降级为编译时常量。max_for_allocation() 仅用于 buffer 分配上界，不作为运算参数。

验收标准：
  1. 违反禁止项「禁止降级为编译时常量」时编译/运行必须失败

REQ-UMK-31 编译管线三层并行化 — Phase 0/2/3 可并行调度

编译管线三层并行化（§7）：Phase 0 Scalar trace 可按 op 并行；Phase 2 融合组之间可并行 emit；Phase 3 ISA lowering 可按融合组并行。并行度由编译器自动调度，保证语义正确性。

验收标准：
  1. 符合 编译管线三层并行化（§7）：Phase 0 Scalar trace 可按 op 并行；Phase 2 融合组之间可并行 emit；Phase 3 ISA lo 的约束定义
