胖 Op 自描述架构 — Op 唯一 IR + 零 context lowering

> SSOT：本文档定义 gllm-kernels JIT 编译器的下一代 Op/VmInstr 自描述架构。取代 03-GRAPH-IR.html §1.2 OpKind 的贫血 enum 设计。

> 动机：当前 OpKind（77 变体）携带的参数不足以让 lowering 自描述，导致 lowering 函数平均 9 参数（prog+op+graph+ctx+input_ptr+weight_ptr+output_ptr+resolver+abi+override）。最近的 audio SIGSEGV 是这种架构缺陷的典型：VmInstr::MemCopy 不自描述条件，依赖 graph-level topology.kv_cache_source 推导，导致 conformer self-attention 错走 KV cache copy 路径写入 NULL。

> 目标：每个 op/instr 自带完整语义，lowering 签名简化为 (prog, op, sess)。

1. Op — 唯一 IR 扁平 enum + Spec struct

所有算子通过 Op 枚举表达（唯一 IR，OpKind 已物理删除）。无参数变体保持 unit，有参数变体携带完整自描述 Spec struct。BUILD 阶段直接构造 Op，不经过 from_op_kind 翻译。

pub enum Op {
    // Attention 类
    MultiHeadAttention(AttentionSpec),
    CachedGqa(CachedGqaSpec),
    MlaAttention(MlaSpec),
    // Norm 类
    RmsNorm(NormSpec), LayerNorm(NormSpec), ValueNorm(NormSpec), HeadRmsNorm(NormSpec),
    // Gemm 类
    Gemm(GemmSpec), GemmBias(GemmSpec), QuantGemm(QuantGemmSpec), FusedRmsNormGemm(FusedNormGemmSpec),
    // Activation 类（无参数保持 unit）
    Silu, Gelu, Tanh, SwiGlu, GeGlu,
    SwiGluClipped { limit: f32 },
    // Structural / MoE / Vision-Audio / Opaque / Telemetry 类
    // ...
}

关键设计原则：

  • 扁平 enum：77 变体保持扁平，不嵌套类别 enum（避免两层 match）
  • Spec struct：横向组合字段（geometry + mask + kv_source + dtype），新增字段不改 match arm
  • 无参数保持 unit：Silu/Gelu 等无配置的 activation 保持 unit 变体

1.1 AttentionSpec — KV cache 来源自描述

AttentionSpec 携带完整 attention 语义，消除对 topology.kv_cache_source 的依赖。

pub struct AttentionSpec {
    pub geometry: AttentionGeometry,    // num_q_heads, num_kv_heads, head_dim
    pub mask: AttentionMask,             // Causal | Full | Sliding(W)
    pub kv_source: KvSource,             // FromTensor | FromCache(layer_idx)
    pub sinks: SinksSpec,                // None | Learnable(sinks_tid)
    pub seq_len: SymDim,
    pub dtype: DType,
    pub load_mode: Option<KvLoadMode>,
}

pub enum KvSource {
    /// K/V 来自当前图的 tensor（self-attention prefill、conformer encoder）。
    /// 不读写持久化 KV cache，zero cache copy overhead。
    FromTensor,
    /// K/V 来自持久化 KV cache（LLM decoder generate loop）。
    FromCache { layer_idx: usize },
}

lowering 时 match op.kv_source 直接决定路径，无需查询 graph topology。

1.2 NormSpec / GemmSpec — dtype 自描述

pub struct NormSpec {
    pub feature_dim: usize,
    pub eps: f32,
    pub dtype: DType,  // 自描述，消除 ctx.dtype
}

pub struct GemmSpec {
    pub m: SymDim, pub n: usize, pub k: usize,
    pub dtype: DType, pub trans_b: bool,
    pub exec_pattern: Option<ExecPattern>,      // 从 ctx.bottleneck_map 内化
    pub parallelism: Option<ParallelismDesc>,    // 从 ctx.parallelism 内化
}

2. CompileSession — 编译会话级共享状态

替代旧 LoweringContext（18 字段）中的编译会话级常量。AbiPtrs 也搬入此处（不再是 lowering 函数的独立参数）。

pub struct CompileSession<'a> {
    pub width: SimdWidth,
    pub sym_map: &'a SymDimSlotMap,
    pub registry: Option<&'a ScalarOpRegistry>,
    pub hook: Option<&'a dyn IsaHook>,
    pub budget: Option<ResourceBudget>,
    pub page_size: usize,
    pub dot_cap: DotProductCap,
    pub kv_elem_bytes: usize,
    pub debug_jit: bool,
    pub abi: AbiPtrs,  // 从 lowering 参数搬入
    pub virtual_activation: Option<&'a VirtualActivationMap>,
    pub virtual_tensor_map: Option<&'a VirtualTensorMap>,
    pub layout: Option<&'a LayoutAssignment>,
    pub batch_ctx_ptr: Option<VRegId>,
}

// 目标 lowering 签名
fn lower_op(prog: &mut VmProgram, op: &Op, sess: &CompileSession) -> Result<(), CompilerError>

LoweringContext 消解：

  • 9 字段搬到 CompileSession（width/sym_map/registry/hook/budget/page_size/dot_cap/kv_elem_bytes/debug_jit）
  • 6 字段内化到各 Op spec（dtype/exec_pattern/bottleneck_map/rope_req/ple_req/dwc_req）
  • 3 字段搬入 CompileSession（virtual_activation/virtual_tensor_map/layout）
  • batch_ctx_ptr 搬入 CompileSession

3. VmInstr — 复合指令自描述

仅复合 VmInstr（MemCopy/VecLoad/VecStore/Gather/Scatter）携带 dtype，纯算术读 sess.dtype。

pub enum VmInstr {
    MemCopy { dst: VRegId, src: VRegId, bytes: usize,
              guard: Option<Predicate>,    // 条件谓词内化
              effect: MemEffect },           // Read/Write/ReadWrite
    VecLoad { dst: VRegId, src: PtrExpr, dtype: DType,
              predicate: Option<Predicate> },
    VecStore { dst: PtrExpr, src: VRegId, dtype: DType },
    // 纯算术：dtype 为 None 时从 sess.dtype 继承（99% 场景），
    // Some 时表示 epilogue 覆盖（混合精度融合：BF16 消费者注入 F32 累加器）
    VecBinOp { dst, a, b, op: VecBinOp, dtype: Option<DType> },
}

4. 迁移路线（10 Phase）

详细执行计划见 §10-15 REQ 规范定义。核心 Phase：

  3. Phase 2：CompileSession 提取
  4. Phase 3：Op 新类型 + 共存适配层
  5. Phase 4-7：按类别迁移（Norm/Activation → Gemm/Quant → Attention/MoE/MLA → Structural/Embed/Vision-Audio）
  6. Phase 8：VmInstr 自描述

  8. Phase 10：OpKind 物理删除 + KvCacheWrite 删除 + KvSource BUILD 阶段策略 + op.inc.rs 重命名 + cache v3

  • 每 Phase 末：cargo test --lib 全绿（gllm-kernels 6847 + gllm ~44000）
  • conformer audio regression test 全程作为 canary
  • Phase 9：LoweringContext / GraphTopologyAnalysis::kv_cache_source / OpKind 零生产代码引用

5.1 Phase 1+2 并行合并策略（architect 建议）

  1. Phase 2 先 merge：ctx.session.* 重命名是机械替换，diff 大但语义简单
  2. Phase 1 rebase 到 Phase 2 之上：Phase 1 新增的 kv_source 逻辑（~50 行）rebase 时把 ctx.xxx 改为 ctx.session.xxx
  3. 冲突解决：dispatch_emit.rs 中以 Phase 2 的 rename 为准，Phase 1 新逻辑用新命名

5.2 KvSource 扩展预留

当前 FromTensor | FromCache(layer_idx) 覆盖 GQA/MHA。未来 MLA 跨层 KV 共享（DeepSeek）可能需要 FromSharedCache { group_id, layer_idx }。当前设计留扩展点，无需立即实现。

5.3 OPCODE_VERSION — JIT cache hash 版本策略

引入 OPCODE_VERSION: u32 常量（Phase 3 引入），每次 Op/VmInstr enum 结构变更 bump 版本号。JIT cache hash 包含此常量，避免手动管理 hash 算法升级。比 Phase 8 整体 hash 策略变更更轻量。

5.4 VecBinOp dtype 融合盲区（architect 修正）

纯算术 VecBinOp 保留 dtype: Option<DType>——None 表示从 sess.dtype 继承（99% 场景），Some 表示 epilogue 覆盖（混合精度融合：BF16 消费者注入 F32 累加器）。避免融合场景下统一读 graph-level dtype 导致宽度错误。

sym_mapstring是ref符号维度槽映射

12. Decision 定义（架构决策）

DEC-FATOP-001: CompileSession 替代 LoweringContext

从 LoweringContext（18 字段）提取编译会话级常量到 CompileSession（13 字段）+ op-local 字段保留在 LoweringContext。

方案选项

  • Option A: 保留 LoweringContext，不提取（拒绝重构）
  • Option B: 完全删除 LoweringContext，所有字段全局化（破坏封装）
  • Option C: 提取 CompileSession + 保留 LoweringContext op-local 字段（chosen）

选定方案

Option C：CompileSession（13 字段）+ LoweringContext（session 引用 + op-local 字段）

理由

编译会话级常量与 op-local 状态分离，符合 SRP（单一职责）。LoweringContext 通过 &session 引用访问全局，零拷贝。

13. Dataflow 定义（编译数据流）

DF-FATOP-001: Op 直接构造 → CompilerOp.op（OpKind 已删除）

BUILD 阶段直接构造 Op，add_op(op: Op) 存储 CompilerOp.op。不再经过 OpKind → from_op_kind 翻译。KvSource 由 features.family 策略选择。

契约

  • from: graph_builder.add_op(op: Op, ...)
  • to: CompilerOp.op = Op
  • transform: Family → KvSource 策略 → Op 直接构造（含完整 Spec struct）
  • direction: sync（编译时一次性）

DF-FATOP-002: Op → lower_op（lowering 入口，唯一 IR）

lowering 路径直接读 op.op，Op 是唯一 IR，零 from_op_kind 调用。

契约

  • from: CompilerOp.op (Op, 不再 Option 缓存)
  • to: lower_op(prog, op: &Op, sess)
  • transform: Op → VmInstr 序列（通过 Op Spec 直接获取参数）
  • direction: sync（编译时一次性）

DF-FATOP-003: VmInstr → x86_64/aarch64/gpu_lower（ISA Lowering）

VmInstr（含 dtype/guard/effect 自描述）→ ISA 特化机器码。

契约

  • from: VmProgram (含 VmInstr 序列)
  • to: Machine Code (x86_64 / aarch64 / PTX / HIP / MSL)
  • transform: VmInstr + DeviceProfile → ISA 指令序列
  • direction: sync（编译时一次性）

14. Module / Interface 定义

MOD-FATOP-001: op 模块（唯一 IR）

Op 枚举 + Spec struct。OpKind enum 已物理删除，from_op_kind 适配层已删除。文件从 op.inc.rs 重命名为 op.inc.rs。

包含

  • Op 枚举定义（77+ 变体，唯一 IR）
  • Spec structs: AttentionSpec/NormSpec/GemmSpec/...
  • OPCODE_VERSION 常量 (v3)
  • key_from_op(op: &Op) → OpKindKey（registry 索引）

对外暴露

  • Op 枚举
  • Op::category(&self) → &'static str
  • add_op(op: Op) / add_op_guarded(op: Op) 单参数 API

依赖

  • CompilerGraph（查询 tensor metadata）
  • CompilerOp（op.inputs/outputs）
  • DType / QuantPrecision（dtype 传播）
  • Family（BUILD 阶段 KvSource 策略选择）

IF-FATOP-001: CompileSession 契约

LoweringContext 通过 &CompileSession 引用访问编译会话级常量。

输入

  • width: SimdWidth
  • sym_map: &SymDimSlotMap
  • registry: Option<&ScalarOpRegistry>
  • hook: Option<&dyn IsaHook>
  • ... 13 字段

输出

LoweringContext.session: &'a CompileSession<'a>（共享引用）

错误类型

  • CompilerError::CodegenViolation（sym_map slot 缺失等）
